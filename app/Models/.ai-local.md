# 🚨 RAMBU BAHAYA: ATURAN DATABASE MODEL 🚨

**BACA SEBELUM MENAMBAHKAN ATAU MENGUBAH MODEL DI DIREKTORI INI!**

1. **DILARANG KERAS MENGGUNAKAN LOGIKA SOFT-DELETE PADA TABEL PEGAWAI!**
   Status Aktif/Non-Aktif pegawai **TIDAK** dieksekusi dengan kolom `deleted_at`. Dilarang bergantung pada pengecekan *SoftDelete* untuk menyaring eksistensi data pegawai!

2. **WAJIB GUNAKAN `kedudukan_hukum_id` SEBAGAI FILTER AKTIVITAS!**
   Pegawai NON-AKTIF selalu ditandai dengan **`kedudukan_hukum_id = '17'`**. Sedangkan Pegawai AKTIF ditandai dengan deret ID lain (01, 02, dsb) atau *NULL*. Segala bentuk query pencarian wajib mematuhi Scope standar untuk memastikan akurasi data.

3. **DILARANG MENGGUNAKAN RELASI FOREIGN KEY `golongan_id` UNTUK KALKULASI PPPK!**
   Namespace ID Golongan dari SIASN memiliki tumpang tindih mutlak (konflik) antara entitas PNS dan PPPK. Anda **WAJIB** merujuk pada nilai mentah dari kolom string **`gol_akhir`** saat melakukan render data UI atau perhitungan transaksi khusus PPPK demi mencegah malfungsi mapping!
