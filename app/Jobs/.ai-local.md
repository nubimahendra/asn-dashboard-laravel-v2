# 🚨 RAMBU BAHAYA: ATURAN QUEUE / JOBS 🚨

**BACA SEBELUM MEMODIFIKASI ATAU MEMBUAT JOB BARU DI DIREKTORI INI!**

1. **DILARANG KERAS MENGGUNAKAN `chunk()`!**
   WAJIB gunakan **`chunkById(500, ...)`** saat iterasi staging table berukuran besar. Penggunaan `chunk()` konvensional AKAN menyebabkan *cursor drift* dan menghilangkan pemrosesan ratusan data penting!

2. **DILARANG MEMBUAT IMPORT BATCH DI DALAM JOB!**
   Objek Import Batch SELALU dicreate **SEBELUM** job di-dispatch (dari Controller/Action terkait). Jangan pernah mencoba melakukan insert entri *batch id* langsung dari dalam class Job!

3. **ISOLASI ERROR PER-RECORD MUTLAK DIPERLUKAN!**
   Error/Exception pada satu record **TIDAK BOLEH MENGHENTIKAN SELURUH JOB**. WAJIB membungkus perulangan iterasi menggunakan block `try-catch`, log pesannya, tandai record tersebut menjadi `is_processed = true` dengan status `failed`, dan **LANJUTKAN** ke iterasi record berikutnya agar bar progres UI tidak *stuck*!
